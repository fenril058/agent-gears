# Grilling 手順書

計画・決定・アイデアのあらゆる側面について、共通理解に達するまでユーザーにインタビューする。
決定木の各分岐を降りながら、決定間の依存関係を1つずつ解決する。

ルール:

- 質問は**一度に1つ**。各問への回答を待ってから次に進む。
  複数の質問を同時にぶつけると相手を混乱させる。
- 各質問には**自分の推奨案**を添える。
- **事実**は環境(ファイルシステム、ツール、git履歴など)を探索して自分で調べる。ユーザーに聞かない。
  **決定**はユーザーのものであり、1つずつ提示して回答を待つ。
- ユーザーが「共通理解に達した」と確認するまで、計画に着手しない。

## 関連スキル

- **spec-ambiguity-audit**: 書かれた仕様書のギャップを安価モデルの素読みで監査する。
  grilling はその対話版で、未決定事項をユーザーと直接解決する。
- **conversation-context-export**: 共通理解に達したら export し、次セッションに決定を引き継ぐ。
