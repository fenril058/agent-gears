# No Neologism (未定義語・勝手造語の点検)

核となる不変則は常時ルール(AGENTS.md / CLAUDE.md)にある:
**定義していない用語・勝手な造語を導入しない。確立した術語を使う。**
この skill はその点検手順を担う。

## 点検手順

1. 文章・回答の中で「概念に名前を与えている語」を洗い出す。特に、
   自分がその場で作った言い回し、比喩を術語のように使っている箇所。
2. 各語について次を確かめる:
   - その分野で**慣用されている語があるか**。あればそれに置き換える
     (意味の近い別語に逃がさない。例「見抜く」を「見分ける」にすり替えない)。
   - 既存語がなく**新語が必要か**。必要なら**初出で定義する**(太字で導入し、
     何を指すかを一文で述べる)。以後はその語で一貫して通す。
   - 単なる飾りで増やした語なら**削る**。
3. 同じ概念を別の語で言い換えていないか。指す対象がぶれる曖昧語
   (「ツール」「文脈」「経路」等)に後退していないか。

## 直し方

- 慣用語があるのに独自語を充てている → 慣用語へ。
- 定義せず使い始めた術語 → 初出位置で定義を補うか、普通の言い方に開く。
- 術語の響きを非術語の場面に流用している → 平易な動詞・名詞で言い直す。

## 併用

日本語の技術文書では `japanese-tech-writing` skill の「視点と語り」
(術語・訳語の慣用語選択、曖昧語への後退の禁止)と合わせて点検する。
